Task:

I am providing newly learned technical knowledge.

Treat this repository as the single source of truth for my personal technical knowledge base.

Your job is NOT simply to append text.

Follow this process:

Step 1

Understand the new content.

Identify:

- technologies
- concepts
- interview topics
- related concepts
- code examples
- diagrams
- images

Step 2

Search the entire repository.

Determine whether this knowledge already exists.

If it exists:

Do NOT create another page.

Merge the knowledge into the existing canonical document.

Step 3

If the topic does not exist:

Create a new document in the correct category.

Use the folder hierarchy defined in KNOWLEDGE_ARCHITECTURE.md.

Step 4

While merging:

Never remove existing information unless it is technically incorrect.

Never lose information.

Prefer merging over replacing.

Preserve existing formatting whenever possible.

Step 5

If the new content belongs in multiple places:

Choose ONE canonical location.

Add cross-links from related pages.

Do not duplicate paragraphs.

Step 6

If diagrams or images are included:

Store them using the project's image strategy.

Do not embed Base64 images.

Step 7

Update navigation if required.

Step 8

At the end provide a report.

Include:

Existing documents updated

New documents created

Duplicate information detected

Cross-links added

Potential improvements

Questions requiring my decision

Never make assumptions when architecture could be affected.

Preserve my knowledge.

Do not rewrite everything simply because it can be written differently.

The goal is to continuously improve a long-term technical knowledge base without creating duplicates or losing information.
